Категория Команда / Директива Что делает Пример
ОПРЕДЕЛЕНИЕ ДАННЫХ (DATA DEFINITION)
Байтыdb / rb1 байт (8 бит). Числа 0-255 или символы.db 0x61, 'A', 0
Словаdw / rw2 байта (16 бит). Числа 0-65535.dw 0xFFFF
Дв. словаdd / rd4 байта (32 бита). Стандарт для адресов.dd 1000000
Кв. словаdq / rq8 байт (64 бита). Для больших вычислений.dq 0xABCDE
ДублированиеdupПовторение значения при определении.db 10 dup (0)
АРИФМЕТИКА И ЛОГИКА
ПересылкаmovКопирует данные из источника в приемник.mov eax, ebx
Сложениеadd / adcСложение / Сложение с учетом флага переноса.add al, 5
Вычитаниеsub / sbbВычитание / Вычитание с заемом.sub esp, 4
Шаг +/-inc / decУвеличить на 1 / Уменьшить на 1 (Быстро).inc edx
Умножениеmul / imulБеззнаковое / Знаковое умножение.mul ecx
Делениеdiv / idivБеззнаковое / Знаковое деление.div ebx
Логикаand / or / xorПобитовые И, ИЛИ, Исключающее ИЛИ.xor eax, eax
Инверсияnot / negПобитовая инверсия / Отрицание (число *-1).neg eax
УПРАВЛЕНИЕ ПОТОКОМ (JUMPS & FLOW)
БезусловныйjmpПереход на метку без условий.jmp @f
СравнениеcmpСравнивает А и Б (вычитает, но не сохраняет).cmp al, 10
Равенствоje / jzПереход, если равно / если результат ноль.je label_exit
Неравенствоjne / jnzПереход, если не равно / не ноль.jnz repeat_loop
Больше/Меньшеjg / jlПереходы для знаковых чисел (Greater/Less).jl error
ЦиклloopУменьшает ECX и прыгает, пока ECX > 0.loop start_loop
Функцииcall / retВызов подпрограммы / Возврат из неё.call print_hex
НИЗКОУРОВНЕВОЕ ВЗАИМОДЕЙСТВИЕ
Портыin / outЧтение из порта / Запись в порт железа.out 0x64, al
Стекpush / popЗатолкнуть в стек / Вытолкнуть из стека.push rax
ПрерыванияintВызов программного прерывания (BIOS/DOS).int 0x10
Флагиstc / clcУстановить / Сбросить флаг переноса (CF).stc
Паузаhlt / nopОстановка процессора / Пустая операция.hlt
ДИРЕКТИВЫ КОМПИЛЯТОРА
ФорматformatТип файла: binary, PE (Windows), ELF.format binary
СмещениеorgАдрес в памяти, куда загрузится код.org 0x7C00
ПовторtimesПовторяет инструкцию или данные.times 510 db 0
СекцииsectionРазделение на код, данные и ресурсы.section '.data'
МакросыmacroСоздание собственных сложных команд.macro invoke proc...
